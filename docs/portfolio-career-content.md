### 9.2 強みを表す文章

```text
既存の仕様やコードを調査し、処理の流れや影響範囲を整理したうえで、設計・実装・テストへつなげることを得意としています。

不明点や問題が発生した際には、関連コード、ログ、データを確認し、原因を切り分けながら解決方法を検討します。必要に応じて小さく試作・検証し、早い段階で認識のずれや技術的な問題を確認することを意識しています。

また、AIを単にコード生成へ使うのではなく、目的、条件、既存仕様、関連コードを共有し、提案内容を自分で確認・修正しながら開発へ活用しています。
```

## 10. AIを活用した開発

## 10.1 現場でのAI活用事例

- 設計内容のレビュー
- 実装・コーディング
- 不具合の原因調査
- 既存システムの仕様確認
- 技術や業務知識の整理と補完

## 10.2 具体的な使い方

```text
確認してほしい設計内容やソースコード、発生しているエラーなどのコンテキストを正しくAIに共有し、目的や条件をプロンプトで具体的に指示しています。

自身の設計内容をAIへ伝え、実装を依頼した後は、自分でコードをレビューします。問題がなければ完了とし、問題がある場合は自分で修正するか、修正内容を具体的に伝えてAIに再対応してもらいます。
```

## 10.3 AI活用で重視していること

- AIへ作業を依頼する前に目的と完了条件を明確にする
- 既存仕様、関連コード、背景情報を不足なく共有する
- 一度に大規模な変更を依頼せず、小さな単位で進める
- 出力されたコードをそのまま採用せず、自分でレビューする
- テストやビルド結果を確認する
- 仕様と異なる変更が含まれていないか確認する
- AIとのやり取りを通じて、技術や業務知識を整理・補完する

## 10.4 ポートフォリオ掲載文

```text
業務および個人開発では、Claude CodeやCodexなどのAIコーディングエージェントを活用しています。設計内容、関連コード、既存仕様、エラー内容などのコンテキストを共有し、目的と条件を明確にしたうえで、調査、実装、レビュー、リファクタリングを依頼しています。

AIの出力は必ず自分で確認し、仕様との整合性、コードの責務、テスト結果をレビューしています。AIへ丸投げするのではなく、自分が設計と品質に責任を持ちながら、開発速度と理解度を高めるための支援ツールとして利用しています。
```
